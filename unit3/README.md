## Core architecture

Two files power everything in this unit:

**`run_agent.py`** — The async agent execution engine.
- `Agent` — a TypedDict with fields: `name`, `model`, `prompt`, `tools`, `kwargs`, `description`
- `run_agent()` — async loop: sends messages to the API, handles tool calls, returns final text
- `as_tool()` — wraps any agent as a callable async function so other agents can invoke it
- `conclude()` — a special tool an agent calls to signal it is done (returns `None` from `run_agent`)

**`agents.py`** — The YAML loader and entry point.
- Loads a YAML file (multiple agent definitions separated by `---`)
- Registers all non-`main` agents as tools in a toolbox
- Finds the `main` agent and runs it
- Handles token usage tracking and final cost reporting

## YAML agent config format

Each YAML document in a config file defines one agent. Here is an annotated example:

```yaml
name: my_agent          # Unique name. 'main' is the entry point agent.
model: gpt-5-mini       # OpenAI model to use
description: |          # Shown as the tool docstring when another agent calls this one
  What this agent does and when to use it
tools:                  # Tools this agent can call
  - talk_to_user        # Built-in: prints a message and waits for user input
  - conclude            # Built-in: ends the agent's turn
  - other_agent_name    # Any other agent defined in the same YAML file
prompt: |               # System prompt for this agent
  You are a helpful assistant that...
kwargs:                 # Passed directly to client.responses.create()
  reasoning:
    effort: minimal     # Options: minimal, medium, high
```

A config file with multiple agents uses `---` as a separator:

```yaml
name: main
tools: [researcher, conclude]
prompt: You are a coordinator. Use the researcher to gather information.
---
name: researcher
description: Searches for information on a given topic.
prompt: You are a research assistant. Find facts about the given topic.
tools: [conclude]
```

## How to run

```bash
# Run with a specific config (from repo root)
python unit3/agents.py unit3/single-agent.yaml

# Pass an initial message to skip the first user prompt
python unit3/agents.py unit3/single-agent.yaml "Hello there"

# Run the deep research pipeline
python unit3/agents.py unit3/deep_research.yaml

# Enable debug logging (prints agent names and timing to stderr)
python unit3/agents.py unit3/deep_research.yaml 2>debug.log
```

## Try it yourself

Work through these in order — each one adds a new concept:

1. **Single agent** — run the minimal pirate chatbot:
   ```bash
   python unit3/agents.py unit3/single-agent.yaml "Ahoy"
   ```

2. **Multi-agent** — run the quotes agent (calls a sub-agent):
   ```bash
   python unit3/agents.py unit3/quotes.yaml
   ```

3. **Modify a YAML** — open `single-agent.yaml`, change the `prompt`, and re-run. No Python changes needed.

4. **Deep research** — run the full 5-agent pipeline on a topic of your choice:
   ```bash
   python unit3/agents.py unit3/deep_research.yaml
   ```
   When prompted, enter a research topic. Expect this to take 1–3 minutes and cost a few cents.

5. **Read `run_agent.py`** — trace the `run_agent()` function to understand exactly how tool calls are handled in the async loop.
